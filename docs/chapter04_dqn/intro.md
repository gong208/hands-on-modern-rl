# 第4章：Value-Based 方法——Q-Learning 与深度 Q 网络

## 本章导读

**核心内容**

- 理解 Q-Learning 如何通过 TD 方法在线更新动作价值，以及表格方法的局限。
- 掌握 DQN 的核心设计——神经网络替代表格、经验回放打破相关性、目标网络稳定训练。
- 从 Double DQN 到 Rainbow，理解 DQN 家族的演进脉络及其对大模型训练的启示。

**核心公式**

$$
Q(s,a) \leftarrow Q(s,a) + \alpha \left[ r + \gamma \max_{a'} Q(s',a') - Q(s,a) \right] \quad \text{（Q-Learning 更新：用 TD 目标修正动作价值）}
$$

$$
\mathcal{L}(\theta) = \mathbb{E}_{(s,a,r,s') \sim \mathcal{D}} \left[ \left( r + \gamma \max_{a'} Q_{\theta^-}(s',a') - Q_\theta(s,a) \right)^2 \right] \quad \text{（DQN 损失函数：用神经网络逼近 Q 值）}
$$

**本章公式的作用**

第 4 章围绕两个核心公式展开：

- **Q-Learning 更新公式**——整条 Value-Based 路线的起点。利用 TD 思想，走一步就能用"当前奖励 + 下一状态的最优价值"来修正当前动作的评分。
- **DQN 损失函数**——把表格换成了神经网络。网络参数 $\theta$ 学习拟合 $Q$ 值，目标网络的参数 $\theta^-$ 冻结不动，保证训练目标不会"追着自己的尾巴跑"。经验回放缓冲区 $\mathcal{D}$ 负责打破连续样本之间的相关性。三个组件合在一起，让深度 Q 网络在 Atari 游戏上实现了人类水平的决策。

2013 年，伦敦的一间小办公室里，一家名叫 DeepMind 的初创公司做了一件让整个 AI 界为之震动的事：他们让一个程序完全从头学会了玩 Atari 游戏。没有人工编写规则，没有告诉程序"球拍应该往哪移"——程序只能看到屏幕上的像素，然后通过不断试错，自己摸索出了得分的方法。从像素到决策，中间没有任何人类知识介入。这个程序用的算法就是 **DQN**（Deep Q-Network），它把第 3 章我们学过的 Q-Learning 从一张小表格搬进了一个神经网络。

第 3 章介绍了路线一的核心思路：学习 **$Q(s,a)$** 来给每个动作打分，然后选分数最高的。简单回顾：$Q(s,a)$ 是动作价值函数，回答"在状态 $s$ 下做动作 $a$，未来一共能拿多少分"。我们还看到了三种计算价值的方法——**DP**（需要环境模型，不实用）、**MC**（要等一整局结束才能更新）、**TD**（走一步就能更新，最实用）。Q-Learning 就是 TD 方法在 $Q$ 值上的直接应用。

但第 3 章只讲了概念层面。在简单的 4×4 GridWorld 里，16 个格子、4 个方向，64 个 $Q$ 值，一张表格即可存储。然而现实世界远不止 16 个格子：

- **CartPole**：状态是 4 维连续向量，理论上无限多个取值。
- **Atari 游戏**：输入是 84×84×4 的像素帧，每帧超过 2.8 万个数值，可能的画面数量远超物理存储的极限。

Q-Learning 的表格在这些高维状态空间面前不再适用。

本章将回答一个关键问题：**当状态空间大到表格装不下时，Q-Learning 还能用吗？** 答案是 DQN——用神经网络替代表格，再配合两个工程技巧，让 Q-Learning 在高维连续状态空间中稳定训练：

1. **经验回放（Experience Replay）**——打破连续帧之间的相关性。
2. **目标网络（Target Network）**——让训练目标在一段时间内保持稳定，避免网络追着自己的尾巴跑。

本章沿着"理论 → 组件 → 观察 → 演进 → 动手 → 项目"的路径展开。先把 DQN 为什么能训练稳定讲清楚，再把完整实现和更大的游戏项目放到章末。
